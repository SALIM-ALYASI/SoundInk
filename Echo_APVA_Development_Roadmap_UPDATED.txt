============================================================
PHASE 02.5 — MERGED FEATURE: ONE PAGE + TWO VOICES + SELF-HEALING
============================================================

Objective:
Merge the “Normal Voice” page and “Cinematic Voice” page into ONE workflow
that generates BOTH outputs, while supporting auto-fix for wrong words and
automatic cleanup of old/temp files.

A) ONE PAGE — DUAL OUTPUT (NORMAL + CINEMATIC)
- Single input page (one text box, one generate button)
- Output 1: Normal voice
- Output 2: Cinematic voice
- Cinematic should be produced via processing chain on top of Normal output
  to keep the SAME voice tone (Normal -> Cinematic Chain -> Cinematic Output)

B) CINEMATIC REORDER / PROCESSING CHAIN
- Create “cinematic presets” (EQ/Comp/Reverb/Normalize…)
- Allow intensity options (light / medium / strong)
- Apply chain consistently per persona/preset

C) SELF-HEALING SEGMENTS (WRONG WORD FIX)
- Split text into segments (sentences/short paragraphs)
- Generate Normal audio per segment and stitch into one file
- If a word is wrong inside a segment:
  - Delete the FULL segment audio
  - Regenerate ONLY that segment (normal)
  - Re-stitch final audio
- This avoids regenerating the entire long audio every time

D) LEXICON MEMORY (SAVE WRONG WORDS UNTIL CORRECT)
- Store corrections in: storage/lexicon/user_fixes.json
- For each wrong word:
  - original_word
  - corrected_word (or pronunciation hint)
  - voice/persona id (optional)
  - count + last_updated
- Before every generation:
  - auto-apply saved corrections to the input text
- Keep words saved until they become correct consistently

E) AUTO CLEANUP (DELETE OLD OUTPUTS & TEMP FILES)
- Use session folder structure:
  storage/sessions/<session_id>/ (normal.wav, normal.mp3, cinematic.mp3, meta.json)
- On new generation:
  - remove previous session outputs (or keep last N sessions)
- Always cleanup temp files:
  storage/tmp/ (cleanup on start + scheduled cleanup)

Deliverable:
Single page pipeline that outputs Normal + Cinematic, supports self-healing
segment regeneration, remembers word corrections, and auto-cleans outputs.
