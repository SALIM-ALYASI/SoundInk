# 🎙️ مشروع SoundInk (APVA)

**SoundInk** هو نظام متطور لتحويل النص إلى كلام (Text-to-Speech) مع التركيز على اللغة العربية، مدعوم بواجهة مستخدم ويب بسيطة ومهام خلفية (Background Workers) لمعالجة الملفات الصوتية بكفاءة عالية.

---

## ✨ المميزات الرئيسية
1. **توليد صوت ذكي**: تحويل النصوص العربية إلى صوت بشري طبيعي باستخدام نماذج متقدمة.
2. **وضع الحوار المتعدد (Dialogue Mode)**: إمكانية كتابة نص يحتوي على عدة شخصيات (مثل: الياسي، سالم، ابتسام)، ويقوم النظام تلقائياً بفصل النص وتخصيص الصوت المناسب لكل شخصية ودمجها في مقطع واحد.
3. **المساحات الصوتية السينمائية (Cinematic Audio Space)**: إضافة فواصل موسيقية سينمائية ذكية، مع خفض مستوى الموسيقى (Ducking) تلقائياً أثناء الحديث لإنتاج مقاطع صوتية احترافية وجاهزة للبودكاست.
4. **معالجة في الخلفية (Celery Workers)**: يتم توليد الصوت ومعالجة الملفات الكبيرة في الخلفية لضمان عدم توقف أو بطء واجهة المستخدم.
5. **خدمة تعمل 24/7 (Background Service)**: إمكانية تشغيل خادم الويب كخدمة مستمرة في خلفية نظام Mac باستخدام `launchctl`.

---

## 🛠️ المتطلبات التقنية (Prerequisites)
- نظام التشغيل: macOS (أو Linux).
- بيئة بايثون: `Conda` (يفضل استخدام البيئة المسماة `apva310`).
- الحزم الأساسية: `FastAPI`, `Celery`, `TTS`, `PyDub` (بالإضافة إلى متطلبات `FFmpeg` مثبته في النظام).

---

## 🚀 كيفية تشغيل المشروع

هنالك طريقتان لتشغيل المشروع: **التشغيل اليدوي** أو **التشغيل كخدمة مستمرة**.

### الطريقة الأولى: التشغيل اليدوي (للتطوير والاستخدام المباشر)
لتشغيل النظام بالكامل، تحتاج إلى تشغيل "خادم الويب" و"مُعالج المهام بالخلفية" (Celery Worker) كلٌ في نافذة موجه أوامر (Terminal) منفصلة.

**1. تشغيل خادم الويب (Web Server):**
افتح Terminal في مجلد المشروع ونفذ السكربت المخصص:
```bash
./run_web.sh
```
*(أو يمكنك تفعيل بيئة الكوندَا يدوياً وتشغيل `python app.py`).*
- سيعمل الموقع على الرابط: [http://localhost:5000](http://localhost:5000)

**2. تشغيل معالج المهام (Celery Worker):**
افتح نافذة Terminal **جديدة** في نفس مجلد المشروع ونفذ:
```bash
./start_worker.sh
```
- هذا السكربت سيقوم بتفعيل بيئة `apva310` وتشغيل Celery في الخلفية لمعالجة مهام توليد الصوت الثقيلة.
- يتم حفظ سجلات هذا المُعالج في ملف `logs/celery.log`.

---

### الطريقة الثانية: التشغيل كخدمة مستمرة 24/7 (Background Service)
إذا أردت أن يعمل خادم الويب دائماً في الخلفية مع بدء تشغيل جهاز الماك (بدون الحاجة لإبقاء نافذة Terminal مفتوحة)، يمكنك استخدام السكربت المخصص لذلك.

**التثبيت والتشغيل:**
```bash
./setup_background_service.sh
```
- هذا السكربت سيقوم بإنشاء الخدمة التلقائية `com.salim.soundink` وتسجيلها في نظام الـ Mac.
- الموقع سيكون متاحاً دائماً عبر [http://localhost:5000](http://localhost:5000).
- السجلات (Logs) ستُحفظ في `logs/web_server.log` و `logs/web_server_err.log`.

**لإيقاف الخدمة المستمرة:**
إذا أردت إيقاف الخدمة لاحقاً، نفذ الأمر التالي:
```bash
launchctl unload ~/Library/LaunchAgents/com.salim.soundink.plist
```

## 📁 هيكل المشروع الأساسي (Project Structure)
- `app.py`: نقطة الدخول الرئيسية لخادم الويب (FastAPI).
- `core/`: يحتوي على إعدادات النظام الأساسية، قاعدة البيانات، والـ (Pipeline) الخاص بمعالجة وتوليد الصوت.
- `api/v1/`: يحتوي على مسارات ومواجهات برمجة التطبيقات (API Routes).
- `services/worker_tasks.py`: يحتوي على مهام Celery للمعالجة في الخلفية.
- `src/speaker/`: يحتوي على مدير التوليد الأساسي `inference_manager.py` المسؤول عن التخاطب مع نماذج الـ TTS.
- `templates/` & `static/`: ملفات واجهة المستخدم (HTML, CSS, JS).
- `data/ref/`: المجلد الذي يحتوي على عينات الصوت المرجعية (Reference Voices) المستخدمة لإنشاء الأصوات.
