خطة بناء وكيل الإخراج الصوتي السينمائي (Echo Director AI) نسخة
خطوة-بخطوة (Stages) — مخصصة لبودكاست “الصدى”

======================================================== 0) الهدف
النهائي (Definition of Done) ——————————————————– - تدخل للنظام: نص
الحلقة (TXT) + إعدادات بسيطة (Style/Mood). - يخرج النظام: ملف صوت نهائي
جاهز للنشر (WAV/MP3) + ملف مشروع (JSON) يحفظ القرارات. - الصوت النهائي
يحتوي على: 1) تقسيم تلقائي إلى فصول/جُمَل 2) فواصل (Pauses) محسوبة
لزيادة الإيقاع والمدة بدون حشو 3) مؤثرات ديناميكية (Echo/Reverb) بحسب
نوع الجملة 4) موسيقى خلفية اختيارية تُخفَّض وترتفع تلقائياً 5) قياس مدة
تقديرية + تقرير بسيط

======================================================== 1) المرحلة
الأولى: إعداد المشروع (Foundation) ——————————————————– 1.1 إنشاء هيكل
مجلدات ثابت: - echo_agent/ - input/ - scripts/ (نصوص الحلقات) -
voice_refs/ (ملفات مرجعية للصوت إن وجدت) - music/ (موسيقى خلفية) -
work/ - tts_raw/ (الصوت الخام قبل المؤثرات) - stt/ (نص + timestamps من
Whisper) - segments/ (قصاصات صوتية لكل جملة/فصل) - mix/ (نتائج مؤقتة) -
output/ - final/ (النسخة النهائية) - reports/ (تقارير JSON/TXT) -
config/ - presets.json (أنماط الإخراج) - lexicon.txt (قائمة كلمات تُعامل
كـ “ضربات”) - run.sh / run.py

1.2 تثبيت الأدوات محلياً (اختيارياً حسب بيئتك): - Whisper (محلي) لتحويل
الصوت إلى نص بتوقيت. - ffmpeg لمعالجة الصوت (قص/دمج/فلترة). - محرك TTS
لديك (XTTS/Piper…).

1.3 تعريف “Preset” أولي للبرنامج: - الاسم: rasis_echo - mood:
calm_mystery - base_pace: 0.95 (أبطأ قليلاً) - echo_strength: dynamic -
pause_policy: cinematic

======================================================== 2) المرحلة
الثانية: إنتاج الصوت الخام (Raw TTS) ——————————————————– 2.1 إدخال نص
الحلقة (input/scripts/episode01.txt)

2.2 تحويل النص إلى صوت خام (بدون مؤثرات قوية): - الهدف: صوت واضح وقابل
للتحليل. - المخرجات: - work/tts_raw/episode01_raw.wav

2.3 شروط النجاح: - الصوت واضح بدون تشويش. - مستوى الصوت (loudness) ليس
منخفضاً جداً ولا مرتفعاً جداً. - لا يوجد قصّ (clipping).

======================================================== 3) المرحلة
الثالثة: استخراج التوقيت (STT + Timestamps) ——————————————————– 3.1
تشغيل Whisper على الصوت الخام: - المدخل: episode01_raw.wav - المخرج
(مثال): - work/stt/episode01.json (segments + timestamps) -
work/stt/episode01.txt (النص الناتج)

3.2 تنظيف النتائج: - إزالة تكرارات Whisper إن وجدت. - توحيد علامات
الترقيم قدر الإمكان.

3.3 شروط النجاح: - timestamps قريبة من النطق الحقيقي. - تقسيم مقبول إلى
جمل/مقاطع.

======================================================== 4) المرحلة
الرابعة: تحليل النص/الصوت (Brain) ——————————————————– 4.1 بناء مصنّف
بسيط للجمل (Rule-based أولاً): يصنّف كل جملة إلى نوع: - QUESTION (؟، هل،
لماذا، كيف…) - STATEMENT (جملة خبرية) - PUNCHLINE (جملة قصيرة قوية:
“الأثر… نادر.”) - TRANSITION (نقل/جسر بين أفكار) - CLOSING (خاتمة/شعار)

4.2 بناء قاموس كلمات “ضربات” (lexicon): أمثلة: - الصدى، الأثر، الهوية،
الضجيج، الصمت، التقنية، زمن، سؤال، حقيقة

4.3 استخراج خصائص لكل جملة: - طول الجملة (عدد الكلمات) - هل تحتوي كلمة
من lexicon؟ - هل هي قصيرة جداً؟ - هل هي سؤال؟ - موقعها في الحلقة
(بداية/وسط/نهاية)

4.4 شروط النجاح: - كل جملة لديها: type + score + مقترح pause/echo.

======================================================== 5) المرحلة
الخامسة: توليد مخطط الإخراج (Director Map) ——————————————————– 5.1 إنشاء
ملف قرارات إخراج (JSON): لكل جملة: - start_time, end_time - text -
type - pause_after_sec - echo_mode: none/light/strong - reverb_mode:
none/room/hall - music_ducking: (0..1) مقدار خفض الموسيقى أثناء الكلام

5.2 سياسة التوقفات (Pause Policy) المقترحة: - بعد QUESTION: 3.0–4.5
ثواني - بعد PUNCHLINE: 4.0–6.0 ثواني + echo light/strong - بعد
TRANSITION: 2.0–3.0 ثواني - بين الفصول: 5.0–8.0 ثواني (مع موسيقى قصيرة)

5.3 سياسة الصدى (Echo Policy) المقترحة: - PUNCHLINE: echo 20–35% -
CLOSING/الشعار: echo 25–45% - STATEMENT العادي: echo 0–15% أو none -
QUESTION: غالباً none (حتى تبقى واضحة)

5.4 شروط النجاح: - director_map.json جاهز ويبدو منطقي عند قراءته.

======================================================== 6) المرحلة
السادسة: تقطيع الصوت إلى مقاطع (Segmentation) ——————————————————– 6.1
باستخدام timestamps، قصّ الصوت الخام إلى مقاطع لكل جملة: -
work/segments/seg_0001.wav - work/segments/seg_0002.wav … إلخ

6.2 إنشاء ملف تجميع (playlist/concat list) للدمج لاحقاً.

6.3 شروط النجاح: - كل مقطع نظيف، لا يقطع الكلمات. - حواف المقطع فيها
fade صغير (5–20ms) لمنع الطقطقة.

======================================================== 7) المرحلة
السابعة: تطبيق المؤثرات على المقاطع (FX Engine) ——————————————————– 7.1
لكل مقطع، طبّق مؤثرات حسب director_map: - Echo (delay+feedback)
خفيف/قوي - Reverb (room/hall) خفيف - EQ بسيط (دفء) - Compression خفيف
لتوحيد المستوى

7.2 إنتاج نسخة FX لكل مقطع: - work/mix/seg_0001_fx.wav …

7.3 شروط النجاح: - المؤثرات لا تخرب وضوح الكلام. - الصدى موجود لكن غير
مزعج.

======================================================== 8) المرحلة
الثامنة: إدخال التوقفات والموسيقى (Timing + Music) ——————————————————–
8.1 إنشاء ملفات صمت (silence) حسب pause_after_sec: -
work/mix/silence_3.0.wav - work/mix/silence_5.0.wav (يمكن توليدها آلياً
عبر ffmpeg)

8.2 إذا توجد موسيقى خلفية: - تحديد مقاطع موسيقى قصيرة للفواصل - تفعيل
ducking: - الموسيقى تنخفض أثناء الكلام وترتفع في الفواصل

8.3 بناء ترتيب نهائي (Timeline): [seg_0001_fx] + [silence] +
[seg_0002_fx] + … + [closing_fx]

8.4 شروط النجاح: - الإيقاع سينمائي. - الفواصل تزيد الوقت والإحساس بدون
ملل.

======================================================== 9) المرحلة
التاسعة: المكس النهائي (Final Mixdown) ——————————————————– 9.1 دمج كل
العناصر إلى ملف واحد: - output/final/episode01_final.wav

9.2 Normalization / Loudness: - توحيد مستوى الصوت النهائي (مثلاً -16
LUFS للبودكاست)

9.3 تصدير نسخة MP3: - output/final/episode01_final.mp3

9.4 شروط النجاح: - الصوت متوازن، واضح، مؤثر. - لا يوجد clipping. - حجم
الملف مناسب.

======================================================== 10) المرحلة
العاشرة: التقارير والتحكم (Reports + Controls) ——————————————————– 10.1
تقرير مدة الحلقة: - إجمالي مدة الكلام - إجمالي مدة الفواصل - مدة
الموسيقى - المدة النهائية

10.2 تقرير قرارات الإخراج (Readable): - جدول بسيط: رقم الجملة / النوع /
pause / echo / ملاحظة

10.3 حفظ كل شيء لسهولة إعادة الإنتاج: -
output/reports/episode01_director_map.json -
output/reports/episode01_report.txt

======================================================== 11) خطة التطوير
المتدرّج (Roadmap) ——————————————————– النسخة V0 (أسبوع): - Rule-based
classification + pauses + echo بسيط - إخراج ملف نهائي واحد

النسخة V1 (أسبوع–أسبوعين): - Presets متعددة (فلسفي/غامض/تقني) - Ducking
للموسيقى تلقائي - واجهة CLI: echo_agent –in episode.txt –preset
rasis_echo

النسخة V2 (أسبوعين+): - تحليل نبرة/طاقة (Energy) من الصوت نفسه - قرارات
pause/echo تعتمد على الأداء الحقيقي - إنشاء نسخة قصيرة تلقائية (promo
60s)

======================================================== 12) اختبارات
الجودة (Quality Checklist) ——————————————————– - وضوح الكلام: ممتاز -
الصدى: جميل وغير مزعج - الإيقاع: لا يوجد بطء ممل - الفواصل: تخدم
المعنى - الموسيقى: لا تطغى على الصوت - المدة: ضمن هدفك (مثلاً 10–14
دقيقة)

======================================================== 13) إعدادات
سريعة مقترحة لبودكاست “الصدى” ——————————————————– - سرعة القراءة (pace):
0.92–0.98 - متوسط pause بعد الجمل العادية: 2.0s - بعد الأسئلة: 3.5s -
بعد الضربات: 5.0s - صدى الشعار (الافتتاح/الخاتمة): 35%

======================================================== ملاحظة أخيرة
——————————————————– ابدأ بـ V0 بسرعة. لا تنتظر الكمال. أول نجاح:
Pipeline يشتغل end-to-end ويطلع ملف نهائي. بعدها نرفع الذكاء خطوة خطوة.
